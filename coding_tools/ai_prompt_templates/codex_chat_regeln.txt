# Zweck
# - Definiert dauerhafte Arbeitsregeln für Codex innerhalb dieses Projekts.
# - Wird zu Beginn eines neuen Codex-Chats geladen.
# - Gilt für alle folgenden Runs.

Projekt
Kino-App (Tagesabschluss)
Flutter / Dart
Repository: kino_bar_app
Branch: master

Grundprinzip
Dieses Projekt arbeitet mit kleinen, kontrollierten Entwicklungs-Runs.

Arbeitsweise
- Änderungen erfolgen ausschließlich über explizite Codex-Run-Prompts.
- Ein Run hat immer genau einen klaren Fokus.
- Keine Nebenbei-Refactors.
- Keine Architekturänderungen ohne Architektur-Run.

Ohne Run-Prompt NICHT erlaubt
- Code ändern
- Dateien umbenennen oder verschieben
- neue Packages hinzufügen
- pubspec ändern
- Plattform- oder Build-Konfiguration ändern
- großflächige Formatierungsänderungen

Ohne Run-Prompt erlaubt
- Verständnisfragen stellen
- kurze Analyse des Codes
- Rückfragen bei unklaren Anforderungen

Technische Leitplanken
- Interne Berechnung weiterhin in Cent.
- Persistenz-Keys dürfen nicht verändert werden.
- UI darf nur im Zielbereich eines Runs geändert werden.
- Keine neuen Dependencies ohne Freigabe.

Run-Regel
Ein Run-Prompt enthält immer:
- Kontext
- Ziel
- Akzeptanzkriterien
- technische Regeln
- Git-Anweisungen

Wenn ein Prompt unklar ist:
→ Rückfragen stellen, keine Änderungen durchführen.

LOCK (WICHTIG)
- Führe keine Änderungen aus.
- Erstelle keinen Code.
- Warte auf den nächsten Codex Run Prompt.

Antwortformat auf diesen Prompt:
Antworte ausschließlich mit: "Bereit. Warte auf Run-Prompt."
