Abschluss (deine Antwort):

- Kurzer Hinweis, wie ich testen kann (3 Bulletpoints).
- flutter analyze muss sauber sein.
- flutter test muss grün sein (falls Tests vorhanden).

GIT (verbindlich – von dir auszuführen):
- Führe die Git-Befehle selbst aus (nicht nur vorschlagen).
- Commit-Message exakt: "Run <NUMMER>: <Kurzbeschreibung>" (Deutsch).
- Sicherheitsregel: Wenn git status NICHT "On branch master" zeigt ODER wenn „detached HEAD“ / unerwartete Deletes / fremde Änderungen auftauchen:
  - STOPPE und gib NUR eine kurze Diagnose + sichere, nicht-destruktive Schritte aus (keine automatischen clean/restore ohne ausdrückliche Anweisung).
